- RabbitMQ는 AMQP라는 표준 프로토콜을 구현한 오픈소스 메시지 브로커
- Producer: 메시지를 생성하고 발송하는 애플리케이션
- Consumer: 메시지를 수신하고 처리하는 애플리케이션
- Broker: Producer, Consumer 사이에서 메시지를 중개하는 소프트웨어. RabbitMQ 서버가 바로 브로커
- Queue: Consumer가 메시지를 가져가기 전까지 메시지를 저장하는 버퍼
- Exchange: 메시지를 어떤 큐로 보낼지 결정하는 '라우팅' 규칙을 가진 컴포넌트
- Binding: Exchange와 Queue를 연결하는 규칙
- RoutingKey: Producer가 Exchange에게 메세지를 보낼 때, 이 메시지가 어떤 경로 큐로 가야할지 알려주는 일종의 '주소' 정보
