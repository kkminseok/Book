- 라우팅 키를 무시하고, 메세지의 헤더에 담긴 여러 속성들을 조합하여 라우팅을 결정
- 헤더 인수(arguments)와 `x-match`
    - 헤더 인수: 헤더의 키-값 쌍의 조건을 정의
    - x-match: 여러 개의 헤더 조건을 어떻게 결합할지를 지정하는 규칙
        - `all`: 모든 헤더 조건이 전부 존재하고 값도 일치해야 메시지가 전달
        - `any`: 헤더 조건이 하나라도 일치하면 메시지 전달
- Topic Exchange보다는 직관적이다. 하지만 해시맵 기반인 Header방식이 좀 더 느리다.



보통 이런걸 쓰는 이유는 A/B테스트나 복잡한 설정을 요할 떄 사용할 듯하다. 또한 동적으로 뭔가 처리할 때 필요해 보인다.
