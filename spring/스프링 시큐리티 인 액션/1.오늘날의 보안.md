- 스프링 시큐리티는 애플리케이션 수준 보안에 속하는 프레임워크. 네트워크 및 데이터 베이스 배포 등에 맞는 보안이 또 따로 있음.
- 보안이 중요한 이유에 대한 설명
    - 뭐 금전적으로 손해볼 수도 있고.. 누군가는 예민하게 좀 더 받아들일 수도 있고..그런 문제
- 애플리케이션을 개발할 때 피해야 하는 일반적인 취약성
    - 인증 취약성
        - 사용자가 악의를 가지고 다른 사람의 기능이나 데이터에 접근할 수 있다는 것.
    - 세션 고정
        - 기존 세션ID가 재사용될 때 유효한 사용자를 가정하여 공격하는 것
    - XSS
        - 스크립트가 뭐 댓글에서 동작하거나 하여 피해를 입히는 것
    - CSRF
    - 주입
        - SQL 주입, XPath주입, 등... 시스템에 손상을 입히는 방법
    - 기밀 데이터 노출
        - 콘솔이나 로그 데이터 등에 정보를 노출하는 형태
    - 메서드 접근 제어 부족
        - 애플리케이션 모든 계층에 권한 부여를 꼼꼼히 하란 의미..
    - 알려진 취약성이 있는 종속성 이용
        - 제목 그대로
